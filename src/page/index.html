🌐 Métodos HTTP: Entendendo as Requisições na Web

Quando navegamos na internet ou interagimos com APIs, usamos os métodos HTTP para comunicar com o back-end. Cada método tem um propósito específico e define a ação que queremos executar no servidor.

Aqui está um resumo dos principais métodos HTTP:

📜 1. GET - Buscar Informações

📌 O que faz?

  • Solicita dados do servidor, mas não altera nada.
  • Como abrir um site ou buscar dados de uma API.

📌 Exemplo:

script.js 
fetch("https://api.exemplo.com/usuarios")
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error("Erro ao buscar usuários:", error));

💡 Usado para:

  • Consultar listas de usuários, produtos, artigos etc.
  • Buscar informações sem modificar nada no servidor.
📜 2. POST - Criar um Novo Registro

📌 O que faz?

  • Envia dados para o servidor para criar um novo recurso.
  • Diferente do GET, pode modificar os dados no servidor.

📌 Exemplo:

script.js 
fetch("https://api.exemplo.com/usuarios", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ nome: "João", email: "joao@email.com" })
})
.then(response => response.json())
.then(data => console.log("Usuário criado:", data))
.catch(error => console.error("Erro ao criar usuário:", error));

💡 Usado para:

  • Criar contas de usuário, posts em um blog, adicionar itens em um banco de dados.
📜 3. PUT - Atualizar um Registro (Completo)

📌 O que faz?

  • Atualiza toda a informação de um recurso existente.
  • Se algum campo estiver ausente, ele pode ser apagado

📌 Exemplo:

script.js 
fetch("https://api.exemplo.com/usuarios/1", {
  method: "PUT",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ nome: "João Silva", email: "joaosilva@email.com" })
})
.then(response => response.json())
.then(data => console.log("Usuário atualizado:", data))
.catch(error => console.error("Erro ao atualizar usuário:", error));

💡 Usado para:

  • Substituir completamente os dados de um usuário, produto, post etc.
📜 4. PATCH - Atualizar um Registro (Parcial)

📌 O que faz?

  • Atualiza apenas campos específicos de um recurso.
  • Diferente do PUT, os campos não enviados continuam inalterados

📌 Exemplo:

script.js 
fetch("https://api.exemplo.com/usuarios/1", {
  method: "PATCH",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ email: "novoemail@email.com" })
})
.then(response => response.json())
.then(data => console.log("E-mail atualizado:", data))
.catch(error => console.error("Erro ao atualizar e-mail:", error));

💡 Usado para:

  • Atualizar somente um campo, como mudar a senha ou e-mail de um usuário.
📜 5. DELETE - Apagar um Registro

📌 O que faz?

  • Remove um recurso do servidor permanentemente.

📌 Exemplo:

script.js 
fetch("https://api.exemplo.com/usuarios/1", {
  method: "DELETE"
})
.then(() => console.log("Usuário deletado!"))
.catch(error => console.error("Erro ao deletar usuário:", error));

💡 Usado para:

  • Apagar um usuário, remover um produto do catálogo, deletar um post etc.
🚀 Conclusão
Método Ação
GET 🔍 Buscar informações (sem modificar nada)
POST ➕ Criar um novo recurso
PUT 💾 Atualizar um recurso por completo
PATCH 🛠️ Atualizar apenas um campo
DELETE 🗑️ Deletar um recurso
🏷 Tipos de Parâmetros em Requisições HTTP

Quando fazemos requisições para APIs, precisamos passar informações para o servidor. Existem três formas principais de enviar esses dados:

📜 1. Query Params (Parâmetros de Consulta)

📌 O que faz?

  • Passados na URL após o ? e separados por &.
  • Usados para filtrar, ordenar ou paginar resultados

📌 Exemplo:

script.js 
fetch("https://api.exemplo.com/usuarios?idade=25&cidade=São Paulo")
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error("Erro:", error));

URL gerada:

arduino 
https://api.exemplo.com/usuarios?idade=25&cidade=São Paulo

💡 Usado para:

  • Filtros em listas (ex: buscar usuários com idade 25).
  • Paginação (?pagina=2&limite=10).
  • Ordenação (?sort=asc).
📜 2. Route Params (Parâmetros de Rota)

📌 O que faz?

  • Fazem parte da URL, sem ?.
  • Usados para identificar um recurso específico.

📌 Exemplo:

script.js 
fetch("https://api.exemplo.com/usuarios/123")
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error("Erro:", error));

URL gerada:

script.js 
https://api.exemplo.com/usuarios/123

💡 Usado para:

  • Buscar um usuário específico (/usuarios/123).
  • Atualizar ou deletar um item (/produtos/45)
📜 3. Request Body (Corpo da Requisição)

📌 O que faz?

  • Não vai na URL! É enviado no corpo da requisição.
  • Usado para enviar dados complexos, como JSON.
  • Exige métodos como POST, PUT ou PATCH.

📌 Exemplo:

script,js 
fetch("https://api.exemplo.com/usuarios", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    nome: "João",
    email: "joao@email.com",
    idade: 25
  })
})
.then(response => response.json())
.then(data => console.log("Usuário criado:", data))
.catch(error => console.error("Erro:", error));

JSON enviado no body:

json 
{
  "nome": "João",
  "email": "joao@email.com",
  "idade": 25
}

💡 Usado para:

  • Criar usuários, enviar formulários, atualizar informações
Tipo de Parâmetro Onde fica? Exemplo
Query Params Na URL após ? /usuarios?idade=25&cidade=SP
Route Params Dentro da URL /usuarios/123
Request Body No corpo da requisição { "nome": "João", "email": "joao@email.com" }
