🌐 Métodos HTTP: Entendendo as Requisições na Web

Quando navegamos na internet ou interagimos com APIs, usamos os métodos HTTP para comunicar com o back-end. Cada método tem um propósito específico e define a ação que queremos executar no servidor.

Aqui está um resumo dos principais métodos HTTP:

1. GET - Buscar Informações

📌 O que faz?

  • Solicita dados do servidor, mas não altera nada.
  • Como abrir um site ou buscar dados de uma API.

📌 Exemplo:

index.js 
const express = require("express")

const app = express()

app.get("/projects", (request, response) => {
  return response.json(["projeto 1", "projeto 2"])
})

💡 Usado para:

  • Consultar listas de usuários, produtos, artigos etc.
  • Buscar informações sem modificar nada no servidor.
2. POST - Criar um Novo Registro

📌 O que faz?

  • Envia dados para o servidor para criar um novo recurso.
  • Diferente do GET, pode modificar os dados no servidor.

📌 Exemplo:

index.js 
app.post("/projects", (request, response) => {
  return response.json(["projeto 1", "projeto 2", "projeto 3"])
})

💡 Usado para:

  • Criar contas de usuário, posts em um blog, adicionar itens em um banco de dados.
3. PUT - Atualizar um Registro (Completo)

📌 O que faz?

  • Atualiza toda a informação de um recurso existente.
  • Se algum campo estiver ausente, ele pode ser apagado

📌 Exemplo:

index.js 
app.put("/projects/:id", (request, response) => {
  return response.json(["projeto 1", "projeto 2", "projeto 3", "projeto 4"])
})

💡 Usado para:

  • Substituir completamente os dados de um usuário, produto, post etc.
4. PATCH - Atualizar um Registro (Parcial)

📌 O que faz?

  • Atualiza apenas campos específicos de um recurso.
  • Diferente do PUT, os campos não enviados continuam inalterados

📌 Exemplo:

index.js 
app.patch("/projects/:id", (request, response) => {
  return response.json(["projeto 1", "projeto 2", "projeto 3", "projeto 4"])
})

💡 Usado para:

  • Atualizar somente um campo, como mudar a senha ou e-mail de um usuário.
5. DELETE - Apagar um Registro

📌 O que faz?

  • Remove um recurso do servidor permanentemente.

📌 Exemplo:

index.js 
app.delete("/projects/:id", (request, response) => {
  return response.json(["projeto 2", "projeto 3", "projeto 4"])
})

💡 Usado para:

  • Apagar um usuário, remover um produto do catálogo, deletar um post etc.
🚀 Conclusão
Método Ação
GET 🔍 Buscar informações (sem modificar nada)
POST ➕ Criar um novo recurso
PUT 💾 Atualizar um recurso por completo
PATCH 🛠️ Atualizar apenas um campo
DELETE 🗑️ Deletar um recurso
🏷 Tipos de Parâmetros em Requisições HTTP

Quando fazemos requisições para APIs, precisamos passar informações para o servidor. Existem três formas principais de enviar esses dados:

1. Query Params (Parâmetros de Consulta)

📌 O que faz?

  • Passados na URL após o ? e separados por &.
  • Usados para filtrar, ordenar ou paginar resultados

📌 Exemplo:

index.js 
app.get("/projects", (request, response) => {
  const { title, owner } = request.query

  console.log(title)
  console.log(owner)

  return response.json(["projeto 1", "projeto 2"])
})

URL gerada:

arduino 
http://localhost:3333/projects?title=React&owner=Guilherme

💡 Usado para:

  • Filtros em listas (ex: Titulo equivalente a "React").
  • Paginação (?title=React&owner=Guilherme).
2. Route Params (Parâmetros de Rota)

📌 O que faz?

  • Fazem parte da URL, sem ?.
  • Usados para identificar um recurso específico.

📌 Exemplo:

index.js 
app.put("/projects/:id", (request, response) => {
  // const params = request.params
  // console.log(params)
  // { id: '1' }

  const { id } = request.params
  console.log(id)
  // 1

  return response.json(["projeto 1", "projeto 2", "projeto 3", "projeto 4"])
})

URL gerada:

script.js 
http://localhost:3333/projects/1?

💡 Usado para:

  • Buscar uma rota específica (/projects/1).
3. Request Body (Corpo da Requisição)

📌 O que faz?

  • Não vai na URL! É enviado no corpo da requisição.
  • Usado para enviar dados complexos, como JSON.
  • Exige métodos como POST, PUT ou PATCH.

📌 Exemplo:

Por padrão o express não interpreta o que vai ser enviado para ele no formato json. Então para falarmos para o express que a nossa API vai receber dados no formato de json, so precisamos adicionar app.use(express.json()).

⚠️ sempre antes da chamada das rotas e logo apos da definição da variavel app!

script,js 
app.use(express.json())

app.post("/projects", (request, response) => {
  const body = request.body
  
  console.log(body)

  return response.json(["projeto 1", "projeto 2", "projeto 3"])
})

JSON enviado no body:

Body => JSON 
{
  "nome": "Guilherme",
  "email": "guilherme@email.com",
  "idade": 27
}

💡 Usado para:

  • Criar usuários, enviar formulários, atualizar informações
🚀 Conclusão
Tipo de Parâmetro Onde Fica? Exemplo Usado Para
Query Params URL após ? /usuarios?idade=25 Filtros, paginação
Route Params Na URL /usuarios/123 Identificar um recurso
Request Body No corpo da requisição { "nome": "João" } Criar ou atualizar dados
Header Params No cabeçalho da requisição "Authorization": "Bearer token123" Autenticação, metadados
Path Params Na URL, dentro de uma hierarquia /lojas/15/produtos/200 Recursos dentro de categorias
Form Data No corpo da requisição formData.append("foto", file) Upload de arquivos
🛑 Middleware: O Interceptador de Requisições 🛑

O middleware é como um segurança que fica no meio do caminho entre a requisição do cliente e a resposta do servidor. Ele pode modificar, bloquear ou encaminhar a requisição antes que ela chegue ao destino final.

O que é um Middleware?

📌 Definição:

Um middleware é uma função que intercepta a requisição antes dela chegar à rota final. Ele pode:

  • Modificar a requisição ou resposta.
  • Bloquear o acesso caso não cumpra algum critério.
  • Registrar logs e métricas.
  • Adicionar informações extras na requisição

📌 Fluxo da Requisição com Middleware:

index.js 
[ Cliente ] → [ Middleware 1 ] → [ Middleware 2 ] → [ Rota Final ]

📜 Exemplo de Middleware no Express.js

📌 Criando um middleware que loga todas as requisições:

index.js 
const express = require("express");
const app = express();

// Middleware que exibe informações da requisição
app.use((req, res, next) => {
  console.log(`[${new Date().toISOString()}] ${req.method} ${req.url}`);
  next(); // Chama o próximo middleware ou rota
});

// Rota de exemplo
app.get("/", (req, res) => {
  res.send("Middleware funcionando!");
});

app.listen(3000, () => console.log("Servidor rodando na porta 3000"));

💡 O que acontece aqui?

  • Toda requisição passa primeiro pelo middleware.
  • Ele exibe a data, método (GET, POST, etc.) e URL da requisição.
  • Depois, o next() permite que a requisição continue até a rota final.

📜 Tipos de Middleware

Os middlewares podem ter diferentes funções, dependendo do que queremos fazer:

1. Middleware Global
  • Aplica-se a todas as requisições.
  • Exemplo: Registro de logs, CORS, parsing de JSON.
index.js 
app.use(express.json()); // Middleware para interpretar JSON
2. Middleware de Rota Específica
  • Funciona apenas em determinadas rotas.
  • Exemplo: Autenticação antes de acessar uma área restrita.

📌 Exemplo:

index.js 
const checkAuth = (req, res, next) => {
  if (!req.headers.authorization) {
    return res.status(401).json({ error: "Acesso negado!" });
  }
  next();
};

app.get("/admin", checkAuth, (req, res) => {
  res.send("Bem-vindo à área administrativa!");
});
3. Middleware de Tratamento de Erros
  • Captura e lida com erros nas requisições.
  • Deve ter quatro parâmetros (err, req, res, next).

📌 Exemplo:

index.js 
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send("Algo deu errado!");
});
🚀 Conclusão
Tipo Descrição Exemplo de Uso
Global Aplica-se a todas as requisições app.use(express.json())
Rota Específica Funciona apenas em algumas rotas app.get("/admin", checkAuth, ...)
Erro Trata erros na aplicação app.use((err, req, res, next) => { ... })
